# High Level Design
Шаблон описания архитектуры ИТ-решения

Структура описания
```
.
├── 1-Problem Statement/        # Описание постановки задачи
│   ├── glossary.md             # Концепции предметной области, сгруппированные по контекстам
│   ├── problem-statement.md    # Краткое описание бизнес-проблемы(возможности)
│   └── use-cases.md            # Варианты использования решения
├── 2-Architecture Vision/      # Общий обзор решения
│   └── vision.md               # Краткое описание вариантов реализации
├── 3-High Level Design/        # Архитектура ИТ-решения
│   ├── uc-realization.md       # Обобщенные сценарии поведения решения
│   ├── context-view.md         # Ландшафт изменения
│   ├── logical-view.md         # Основные элементы решения и отношения между ними
│   ├── interaction-view.md     # Интеграционная архитектура 
│   └── deployment-view.md      # Описание развертывания решения
├── 4-ADRs/                     # Каталог архитектурных решений
│   └── 0001-adr-name.md        # Запись архитектурного решения
└── 5-Enterprise Landscape/     # Окружение
    ├── services.md             # Используем корпоративные сервисы
    └── tech-radar.json         # Набор технологий
```
