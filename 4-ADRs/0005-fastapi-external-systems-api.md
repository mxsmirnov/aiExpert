# ADR-0005: FastAPI для API внешних систем (ACT-005)

**Статус:** Черновик  
**Дата:** 2026-03-20  
**Автор:** [Имя, роль/ID ACT-xxx]  
**Связанные артефакты:** UC-001, UC-002, UC-003 (инициатор ACT-005); ACT-005; [uc-realization.md](../3-High%20Level%20Design/uc-realization.md); ADR-0004, ADR-0006

## 1. Контекст
- **Инициатива/проблема:** Внешние информационные системы (**ACT-005**) должны создавать ADR Request (UC-001), выполнять структурированные и произвольные запросы (UC-002, UC-003). Им нужен **стабильный, версионируемый** контракт M2M, отличный от браузерных сессий сотрудников.
- **Драйверы:** Безопасность интеграций; независимая эволюция внешнего API; автогенерация OpenAPI и контрактного тестирования.
- **Ограничения:** Различные модели доверия (API key, OAuth2 client credentials, mTLS); требования аудита вызовов от внешних систем.
- **Допущения:** Трафик проходит через API Gateway (ADR-0004); доменная логика выполняется в ядре Python (ADR-0006) — интеграционный FastAPI слой может быть отдельным деплоем или логически изолированным приложением/роутером в том же репозитории.


## 2. Проблема и целевой результат
- **Проблема:** Смешение cookie/JWT для сотрудников и M2M-доступа внешних систем в одном поверхности API повышает риск ошибок конфигурации и усложняет модель угроз.
- **Цели/Success Criteria:**
  - Опубликованный **OpenAPI** для внешнего контура;
  - Политика версий (например, `/v1/…`) и deprecate window;
  - Идентификация вызывающей системы в логах и аудите.

## 3. Решение
- **Выбранный вариант:** Реализовать **интеграционный HTTP API на FastAPI** для сценариев **ACT-005** (создание запросов, структурированные/произвольные запросы к архитектурным артефактам согласно UC).
- **Что будет сделано:**
  1. Выделить набор эндпоинтов и схем Pydantic для внешних систем.
  2. Подключить аутентификацию M2M на perimeter (Gateway) и/или в приложении; согласовать scopes.
  3. Делегировать бизнес-операции ядру (сервисный слой ADR-0006), избегая дублирования правил.
  4. При необходимости добавить webhooks/callbacks для асинхронных результатов — отдельной спецификацией.
- **Границы решения:** Не включает UI Reflex (ADR-0007). Не определяет детали оркестрации ADR-Worker (ADR-0002, 0003), кроме триггеров, согласованных с ядром.
- **Затронутые элементы архитектуры:** [API-external], [SRV-external-api]; UC-001–003; ACT-005.

## 4. Рационализация
- **Причины выбора:** FastAPI даёт быстрый REST, валидацию и OpenAPI из коробки; согласован с ADR-0006 (Python).
- **Компромиссы:** Нужна дисциплина не смешивать внешние и внутренние роуты без явного разграничения.

## 5. Альтернативы
1. **GraphQL для внешних систем** — гибкость, выше сложность контракта и кэширования. **Отклонено** на первом этапе в пользу REST/OpenAPI.
2. **Прямой доступ внешних систем к БД или к внутренним gRPC без публичного слоя** — неприемлемо по безопасности. **Отклонено**.

## 6. Последствия

### 6.1 Технические последствия
- Отдельные тесты контракта (schemathesis, consumer-driven); миграции версий API.

### 6.2 Организационные последствия
- Onboarding внешних команд: выдача учётных данных, sandbox.

### 6.3 Эксплуатация и безопасность
- Rate limit по client_id; аудит; ротация ключей; мониторинг ошибок валидации и 401/403.

### 6.4 Риски
- Несовместимые изменения в v1. **Митигирующие меры:** версионирование, контрактные тесты, changelog для интеграторов.

## 7. План внедрения и валидации
- **План:** OpenAPI v1 → sandbox → пилот с одной внешней системой → прод.
- **Проверка:** контрактные тесты; пенетрейшн периметра для M2M.

## 8. Примечания и связанные решения
- **ADR-0004** — входная точка.
- **ADR-0006** — доменное ядро, которому делегирует интеграционный слой.
