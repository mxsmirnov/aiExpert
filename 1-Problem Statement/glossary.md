# Глоссарий решения aiExpert

Этот файл содержит единый язык (Ubiquitous Language), используемый в предметной области "Agentic RAG" (Retrieval-Augmented Generation с агентным подходом). В соответствии с подходом DDD (Domain Driven Design), термины сгруппированы по ограниченным контекстам (Bounded Contexts), что позволяет избежать двусмысленности и обеспечить согласованное понимание внутри каждой функциональной области.

---

## 1. Ограниченный контекст: Обработка запроса (Query Processing)

*В этом контексте описывается путь пользовательского запроса от момента его получения до момента, когда он готов к выполнению поисковым агентом.*


| Термин                                         | Определение                                                                                                                                                                                                                                         |
| ---------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Пользовательский запрос (User Query)**       | Исходный ввод, полученный от пользователя. Может быть задан на естественном языке, в виде вопроса, команды или фрагмента данных.                                                                                                                    |
| **Намерение (Intent)**                         | Цель или задача, которую пользователь хочет решить с помощью своего запроса. Классификация намерения (например, "поиск факта", "суммаризация", "сравнение") определяет дальнейшую стратегию обработки.                                              |
| **Сущность (Entity)**                          | Ключевой объект или понятие, извлеченное из запроса, имеющее значение для выполнения задачи. (Примеры: даты, имена, названия компаний, технические параметры).                                                                                      |
| **План (Plan)**                                | Пошаговая инструкция (sequence of steps), генерируемая агентом на основе намерения и сущностей. План описывает, какие инструменты вызывать, в каком порядке и как обрабатывать результаты для ответа на запрос.                                     |
| **Разрешение ссылок (Coreference Resolution)** | Процесс, в котором агент определяет, к чему относятся местоимения или неявные ссылки в запросе (особенно в многоходовых диалогах). (Пример: "Какая у нее температура плавления?" -> определение, что "она" относится к ранее упомянутому веществу). |


## 2. Ограниченный контекст: Оркестрация агентов (Agent Orchestration)

*Этот контекст описывает структуру и взаимодействие агентов для выполнения плана и получения информации из внешних источников.*


| Термин                          | Определение                                                                                                                                                                                                                                                    |
| ------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Агент (Agent)**               | Автономный программный компонент, который принимает решения и выполняет действия для достижения цели. В Agentic RAG агент отвечает за планирование, выбор инструментов и интерпретацию результатов.                                                            |
| **Инструмент (Tool)**           | Функция или API, которые агент может вызывать для взаимодействия с внешним миром. Включает поисковые системы, калькуляторы, вызовы к базе знаний, выполнение кода и т.д.                                                                                       |
| **Пайплайн RAG (RAG Pipeline)** | Процесс, состоящий из этапов: получение запроса -> поиск релевантных документов -> дополнение промпта (Augmentation) -> генерация ответа (Generation). В Agentic RAG этот пайплайн динамически управляется агентом.                                            |
| **Маршрутизация (Routing)**     | Решение агента о том, к какому именно индексу, источнику данных или под-агенту обратиться на основе текущего контекста и намерения. (Пример: маршрутизация запроса о финансах к векторному индексу с финансовыми отчетами, а запроса о погоде — к API погоды). |
| **Под-агент (Sub-agent)**       | Специализированный агент, вызываемый главным (супервизором) для выполнения конкретной подзадачи. Позволяет реализовывать иерархическую структуру и разделение ответственности.                                                                                 |
| **Наблюдение (Observation)**    | Результат, возвращенный после выполнения агентом вызова инструмента. Агент анализирует наблюдение, чтобы решить, завершена ли задача или нужно сделать следующий шаг.                                                                                          |


## 3. Ограниченный контекст: База знаний и поиск (Knowledge Base & Retrieval)

*Этот контекст описывает, как информация хранится, индексируется и извлекается для использования агентами.*


| Термин                                  | Определение                                                                                                                                                                                        |
| --------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Векторное представление (Embedding)** | Числовое представление (вектор) текста или другого типа данных (изображения, аудио), которое отражает его семантический смысл. Используется для поиска по сходству.                                |
| **Векторное хранилище (Vector Store)**  | База данных, специализированная на хранении векторных представлений и эффективном поиске ближайших соседей (k-NN) среди них.                                                                       |
| **Чанк (Chunk)**                        | Фрагмент документа (обычно несколько предложений или абзацев), который индексируется и хранится в векторном хранилище. Размер и стратегия разбиения на чанки критически влияют на качество поиска. |
| **Метаданные (Metadata)**               | Структурированная информация, привязанная к чанку (например, источник, автор, дата, тип документа). Используется для фильтрации и уточнения поиска.                                                |
| **Поисковый ранг (Retrieval Score)**    | Числовая метрика, показывающая степень сходства между запросом и найденным чанком в векторном пространстве.                                                                                        |
| **Гибридный поиск (Hybrid Search)**     | Метод поиска, комбинирующий семантический поиск (по векторам) с традиционным полнотекстовым поиском (по ключевым словам) для повышения релевантности результатов.                                  |
| **Граф знаний (Knowledge Graph)**       | Структура данных, хранящая информацию в виде сущностей (узлов) и связей (ребер) между ними. Позволяет агентам выполнять логические выводы и находить сложные взаимосвязи.                          |


## 4. Ограниченный контекст: Генерация ответа и достоверность (Response Generation & Grounding)

*Этот контекст описывает финальную стадию создания ответа и обеспечения его фактической точности на основе найденной информации.*


| Термин                                         | Определение                                                                                                                                                                                                                                               |
| ---------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Большая языковая модель (LLM)**              | Глубокая нейронная сеть, способная понимать и генерировать текст. Является "мозгом" агента и генератором финальных ответов.                                                                                                                               |
| **Контекст (Context Window)**                  | Максимальный объем текста (в токенах), который LLM может "помнить" и обрабатывать за один раз. Включает историю диалога, найденные документы и промпт.                                                                                                    |
| **Промпт (Prompt)**                            | Инструкция, передаваемая LLM, которая направляет ее поведение. Включает системный промпт (определяющий роль агента) и промпт пользователя (содержащий задачу и контекст).                                                                                 |
| **Дополнение (Augmentation)**                  | Этап, на котором найденные релевантные документы (контекст) вставляются в промпт вместе с исходным запросом пользователя.                                                                                                                                 |
| **Обоснование (Grounding)**                    | Процесс, обеспечивающий, чтобы ответ LLM был основан исключительно на предоставленных документах (или других проверенных источниках), а не на внутренних "знаниях" модели (параметрической памяти). Это ключевой принцип RAG для борьбы с галлюцинациями. |
| **Галлюцинация (Hallucination)**               | Генерация моделью фактов или утверждений, которые не соответствуют действительности и не содержатся в предоставленном ей контексте (grounding data).                                                                                                      |
| **Цитата (Citation)**                          | Ссылка на исходный документ или чанк, из которого была взята информация для конкретного утверждения в ответе. Критически важна для проверяемости (verifiability) ответа агента.                                                                           |
| **Оценка релевантности (Re-rank / Reranking)** | Процесс повторного ранжирования изначально найденных чанков с помощью более точной (но часто более медленной) модели для отбора только самых релевантных фрагментов перед передачей их в LLM.                                                             |
