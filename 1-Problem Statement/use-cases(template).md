#### Основной сценарий (Happy Path)
*Идеальный поток событий без ошибок.*

| Шаг | Действие |
| :--- | :--- |
| 1 | Пользователь переходит к оформлению заказа. |
| 2 | Система отображает список товаров в корзине и предлагает выбрать способ доставки и оплаты. |
| 3 | Пользователь заполняет адрес доставки и выбирает способ оплаты "Банковская карта". |
| 4 | Система перенаправляет пользователя на страницу платежного шлюза. |
| 5 | Пользователь вводит данные карты и подтверждает платеж. |
| 6 | Платежный шлюз возвращает системе статус "Успешно". |
| 7 | Система создает заказ со статусом "Оплачен" и отображает пользователю страницу подтверждения. |
| 8 | Система отправляет письмо с деталями заказа на email пользователя. |

#### Альтернативные сценарии / Исключения
*Что может пойти не так?*

| Шаг | Условие | Действие |
| :--- | :--- | :--- |
| 4a | **Ошибка платежа** | Если платежный шлюз возвращает ошибку (недостаточно средств, неверный CVV), система возвращает пользователя на шаг 2 с сообщением об ошибке. Заказ не создается. |
| 4b | **Таймаут шлюза** | Если платежный шлюз не отвечает более 30 секунд, система показывает пользователю сообщение о технических проблемах и предлагает повторить попытку позже. Заказ сохраняется как "Черновик". |
