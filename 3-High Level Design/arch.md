# Архитектура системы aiExpert (C4 Container)

Текстовое описание соответствует диаграмме контейнеров **C4 Container Diagram — Система aiExpert** (anti-example). Ниже — роли, внешние системы, контейнеры, хранилища и взаимосвязи в виде таблиц и списков.

---

## Обзор

- **Система:** aiExpert — платформа для запросов на архитектурные решения, жизненного цикла ADR, справочников ABB, отчётности и работы ИИ-агента ADR-Worker.
- **Точка входа для пользователей:** веб-приложение (SPA) за **API Gateway**.
- **Точка входа для интеграций:** внешняя система → API Gateway; агент → MCP-источник.

---

## Участники (люди)

| Роль | Название | Описание |
|------|----------|----------|
| Инициатор | Сотрудник | Инициирует запросы на архитектурные решения |
| Эксперт | Архитектор | Ответственный за подготовку ADR |
| Контроль | Менеджер | Контролирует работу архитекторов |
| Администрирование | Администратор | Управляет системой и справочниками |

**Взаимодействие с системой:**

- Сотрудник → веб-приложение: создание запросов (UC-001, UC-002, UC-003), HTTPS.
- Архитектор → веб-приложение: работа с ADR (UC-004), HTTPS.
- Менеджер → веб-приложение: просмотр отчётов (UC-006), HTTPS.
- Администратор → веб-приложение: справочники и команда (UC-007, UC-008), HTTPS.

---

## Внешние системы

| Идентификатор | Название | Назначение |
|---------------|----------|------------|
| ACT-005 | Внешняя система | Инициирует запросы через API |
| ACT-006 | MCP-источник | Внешний провайдер данных и инструментов (LLM, Git, Jira и т.п.) |

**Связи:**

- Внешняя система → API Gateway: запросы (UC-001, UC-002, UC-003), HTTPS/REST.
- ИИ-агент ADR-Worker → MCP-источник: MCP-инструменты (UC-005), MCP-протокол/HTTPS.

## Маршрутизация через API Gateway

API Gateway маршрутизирует запросы (**REST/HTTPS**) к:

- сервису запросов;
- сервису ADR;
- сервису ABB;
- сервису отчётности;
- сервису справочников;
- сервису команды;
- оркестратору ИИ-агентов (**«Отправляет команды агенту»**).

## Краткий поток по ролям

1. **Пользователь** открывает SPA → трафик идёт через **API Gateway** к REST-сервисам.
2. **Внешняя система** обращается к **API Gateway** для сценариев UC-001–003.
3. **Запросы и ADR** при необходимости инициируют работу через **оркестратор** → **ADR-Worker**, который использует **MCP-источник** и обновляет доменные сервисы и свою БД.
4. **Отчётность** читает агрегированные данные из операционных сервисов и пишет/читает аналитическое хранилище.
